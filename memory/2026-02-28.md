### 飞书 Docx 表格 API 踩坑记录 ⚠️
**背景**：用户要求在 Agent & LLM 调研文档里用 markdown 表格表达 30 个案例

**踩坑1**：`feishu_doc append` markdown 语法不支持 `|table|` 格式 → 400 报错

**踩坑2**：飞书 docx block API 创建 table（block_type: 31）的正确参数：
```json
{
  "block_type": 31,
  "table": {
    "rows": N,
    "columns": M,
    "property": {
      "row_size": N,
      "column_size": M
    }
  }
}
```
- `property` 字段必须有 `row_size` 和 `column_size`（否则 99992402 field validation error）
- `column_width`、`border_visible` 等字段会触发 1770001 invalid param（原因未知）

**踩坑3**：**单次创建表格最多 9 行**（9 行 × 5 列 = 45 cells，接近 API 批量上限）

**踩坑4**：**已创建的 table 不能追加行**（1770028 "block not support create children"）

**解决方案**：按分类拆成多个小表格，每个 ≤ 9 行
- 最终成功：6 个分类表格全部写入，30 行数据全部填充
- 文档：https://bytedance.sg.larkoffice.com/docx/RiKdduWH6oJ3Wwxt4jslF7M3gqe

**删除 block 正确 API**：
```
DELETE /open-apis/docx/v1/documents/{doc_id}/blocks/{parent_id}/children/batch_delete
Body: {"start_index": N, "end_index": N+1}
```
（注意：之前用 `/batch_delete` 路径是404，要用 `/children/batch_delete`）

1. **飞书 Docx table 有 9 行上限**：每次创建不超过 9 行，需多表格分批
