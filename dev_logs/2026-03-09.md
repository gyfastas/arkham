## 前端战斗轮测试反馈

### Bug #1: 缺少交战选项 + 无法攻击未交战敌人
- **现象**: 闪避敌人后，敌人在同一地点但未交战，无法再对其发动攻击
- **原因**: 前端 Fight 只查找 `threat_area`（已交战敌人），忽略了地点上的未交战敌人
- **规则**: 调查员可以攻击同一地点的任何敌人（未交战的需先自动交战）
- **修复**:
  - 前端增加 Engage 按钮
  - Fight 目标选择展示所有地点敌人（交战+未交战）
  - 后端 FIGHT 行动对未交战敌人自动先交战

### Bug #2: 技能卡无法投入检定
- **现象**: 技能检定直接结算，没有投入手牌的交互步骤
- **原因**: 前端没有实现 ST.2（投入卡牌）步骤
- **规则**: 在技能检定中，可以从手牌投入技能卡（以及带匹配图标的其他卡），图标加到检定值上
- **修复**:
  - 前端增加「投入卡牌」弹窗（战斗/调查/闪避时触发）
  - 展示手牌中可投入的卡牌及其技能图标
  - 将选中的卡牌 ID 随行动一起发送给后端

## Refactor: 调查员卡与玩家手牌分离

### 问题
- `CardData` 同时承载调查员和玩家手牌数据，字段混杂
- 调查员特有的经验值、构筑要求、签名卡/弱点、战役创伤无法表达

### 新模型: `backend/models/investigator.py`
- `InvestigatorCard`: 调查员专用数据类，包含四维、能力、远古印记、构筑要求、战役状态(XP/创伤)
- `DeckRequirement`: 构筑要求（牌组大小、允许阵营、等级限制、必需卡、弱点）
- `effective_health/effective_sanity`: 考虑创伤后的实际生命/理智
- `InvestigatorState` 新增 `investigator_card` 字段，优先使用新模型
- `Game.add_investigator()` 支持传入 `InvestigatorCard`，自动生成 `CardData` shim 保持向后兼容

### 迁移
- `server_daisy.py`: 使用 `InvestigatorCard` + `DeckRequirement`（含签名卡/弱点声明）
- `server_full.py`: 使用 `InvestigatorCard`
- 所有291条旧测试继续通过

## Feature: 多幕密谋/事件系统

### 新模型: `backend/models/scenario.py`
- `AgendaCard`: 单张密谋卡（doom_threshold, advance_effects, branch）
- `ActCard`: 单张事件卡（clue_threshold / 特殊条件, advance_effects, branch）
- `AdvanceEffect`: 推进效果（spawn_enemy, add_location, deal_damage, branch, resolution等）
- `Branch`: 分支选择（condition, true_target, false_target）
- `Resolution`: 剧本结局（XP奖励、创伤）
- `AdvanceEffectType`: 枚举所有推进效果类型

### ScenarioState 扩展
- 新增 `agenda_cards`, `act_cards`, `resolutions` 字典
- 新增 `current_agenda`, `current_act`, `effective_doom_threshold` 属性
- `MythosPhase._check_doom_threshold()` 改用 `effective_doom_threshold`
- 向后兼容：无 agenda_cards 时 fallback 到 `doom_threshold`
